Module Federation embed (alternative to the script loader)

This page is itself a small React app. Instead of the <script> loader every other page on this site uses, it dynamically loads the WebChat widget's Module Federation remote directly — the same @module-federation/runtime API RiptaCRM's own Host shell uses to load admin-configured custom menu items at runtime. This path only makes sense for a customer site that's itself a compatible React + Module Federation app.
